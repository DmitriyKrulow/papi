# 📋 ПОЛНАЯ ИНСТРУКЦИЯ ПО ЗАПУСКУ СИСТЕМЫ АВТОРИЗАЦИИ

## 🔧 ШАГ 1: Установка зависимостей

### Вариант А: Автоматическая установка (Windows)
```bash
setup.bat
```

### Вариант Б: Ручная установка

#### Backend:
```bash
cd backend
pip install -r requirements.txt
mkdir -p backend/data
```

#### Frontend (отдельно):
```bash
cd frontend
npm install
```

---

## ▶️ ШАГ 2: Запуск Backend

### 1. Перейдите в папку backend:
```bash
cd F:\Code\papi\backend
```

### 2. Запустите backend:
```bash
python -m src.infrastructure.main
```

### 3. Проверьте, что backend запустился:
Откройте браузер и перейдите на:
```
http://localhost:8888/health
```

Должен отобразиться ответ:
```json
{
  "status": "ok",
  "service": "papi"
}
```

### 4. Документация API:
```
http://localhost:8888/docs
```

---

## ▶️ ШАГ 3: Запуск Frontend

### 1. Откройте новую консоль (не закрывайте предыдущую)
### 2. Перейдите в папку frontend:
```bash
cd F:\Code\papi\frontend
```

### 3. Запустите frontend:
```bash
npm run dev
```

### 4. Проверьте, что frontend запустился:
Откройте браузер и перейдите на:
```
http://localhost:5173
```

Должна открыться главная страница PAPI.

---

## 🔐 ШАГ 4: Первый вход в систему

### 1. Перейдите на страницу входа:
```
http://localhost:5173/login
```

### 2. Используйте учетные данные начального администратора:
```
Username: admin
Password: admin123
```

### 3. Нажмите "Войти"

Если все работает, вы будете перенаправлены на страницу активов.

---

## 👨‍💼 ШАГ 5: Доступ к админ-панели

### После входа как администратор:

1. Откройте админ-панель:
   - Нажмите на ссылку "⚙️ Админка" в навигационной панели, или
   - Перейдите по адресу: `http://localhost:5173/admin`

2. Вы увидите список всех пользователей

3. Функции админ-панели:
   - Просмотр всех пользователей
   - Смена ролей (user ↔ admin)
   - Удаление пользователей

## 🧪 ШАГ 6: Тестирование функционала

### Тест 1: Регистрация нового пользователя

1. Перейдите на страницу регистрации:
   ```
   http://localhost:5173/register
   ```

2. Заполните форму:
   ```
   Имя пользователя: testuser
   Email: test@example.com
   Пароль: test123
   Подтверждение пароля: test123
   ```

3. Нажмите "Зарегистрироваться"

4. Вы должны быть перенаправлены на страницу входа

5. Попробуйте войти с новыми учетными данными

### Тест 2: Обновление профиля

1. Войдите в систему
2. Перейдите на страницу профиля:
   ```
   http://localhost:5173/profile
   ```

3. Обновите свои данные
4. Нажмите "Сохранить"

### Тест 3: Смена пароля

1. На странице профиля перейдите в раздел "Безопасность"
2. Заполните форму:
   ```
   Старый пароль: текущий пароль
   Новый пароль: новый_пароль
   Подтвердите новый пароль: новый_пароль
   ```

3. Нажмите "Сохранить"
4. Выйдите и войдите заново с новым паролем

## 🔍 ПРОВЕРКА РАБОТЫ

### Проверка backend через curl:

#### 1. Проверка состояния:
```bash
curl http://localhost:8888/health
```

#### 2. Регистрация:
```bash
curl -X POST http://localhost:8888/api/auth/register ^
  -H "Content-Type: application/json" ^
  -d "{\"username\":\"test\",\"email\":\"test@test.com\",\"password\":\"test123\"}"
```

#### 3. Вход:
```bash
curl -X POST http://localhost:8888/api/auth/login ^
  -H "Content-Type: application/json" ^
  -d "{\"username\":\"admin\",\"password\":\"admin123\"}"
```

Ответ должен содержать `access_token`.

#### 4. Получение профиля:
```bash
curl http://localhost:8888/api/auth/me ^
  -H "Authorization: Bearer YOUR_TOKEN"
```

#### 5. Админ-панель:
```bash
curl http://localhost:8888/api/admin/users ^
  -H "Authorization: Bearer YOUR_TOKEN"
```

## 🐛 Устранение проблем

### Проблема 1: Backend не запускается

**Симптом:** Ошибка "Module not found"

**Решение:**
```bash
cd F:\Code\papi\backend
pip install -r requirements.txt
```

### Проблема 2: 401 Unauthorized при входе

**Симптом:** При входе возвращается ошибка 401

**Решение:**
1. Убедитесь, что backend запущен на порту 8888
2. Проверьте, что вы используете правильные учетные данные
3. Попробуйте перезапустить backend

**Стандартные учетные данные:**
```
Username: admin
Password: admin123
```

### Проблема 3: Frontend не подключается к backend

**Симптом:** Ошибки CORS в консоли браузера

**Решение:**
1. Убедитесь, что backend запущен
2. Проверьте, что backend разрешает CORS для `localhost:5173`
3. Проверьте настройки в `main.py` (строки 29-38)

### Проблема 4: Нет доступа к админ-панели

**Симптом:** Редирект на главную страницу или 403

**Решение:**
1. Убедитесь, что вы вошли как `admin`
2. Проверьте поле `role` в таблице пользователей
3. Перезапустите backend (пересоздаст администратора)
